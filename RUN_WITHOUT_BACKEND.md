# Запуск приложения без Backend

Приложение готово к запуску в демо-режиме без backend. Firebase и API подключения сделаны опциональными.

## Быстрый запуск

1. **Установите зависимости:**
```bash
flutter pub get
```

2. **Для iOS (если нужно):**
```bash
cd ios
pod install
cd ..
```

3. **Запустите приложение:**
```bash
flutter run
```

## Что будет работать

✅ Все экраны UI (логин, регистрация, список чатов, экран чата)  
✅ Навигация между экранами  
✅ UI компоненты и анимации  
✅ Тема приложения (светлая/темная)  

## Что не будет работать (без backend)

❌ Реальная аутентификация  
❌ Отправка/получение сообщений  
❌ WebSocket подключение  
❌ Push-уведомления (без Firebase)  
❌ Google OAuth (без настроенного Client ID)  

## Ошибки при запуске

Если видите ошибки связанные с Firebase - это нормально. Приложение продолжит работу в демо-режиме.

Ошибки будут в консоли, но не остановят приложение:
- `Ошибка инициализации Firebase` - нормально без Firebase файлов
- `Push-уведомления недоступны` - нормально без Firebase
- `API URLs не настроены` - нормально для демо

## Просмотр UI

Вы сможете увидеть:
1. **Splash Screen** - экран загрузки
2. **Login Screen** - экран входа (можно попробовать ввести данные, но авторизация не сработает)
3. **Register Screen** - экран регистрации
4. **Chats List** - список чатов (будет пустым без backend)
5. **Chat Screen** - экран чата (можно открыть, но сообщения не будут отправляться)
6. **Group Create/Join** - экраны создания/присоединения к группам

## Запуск на конкретном устройстве

```bash
# Список доступных устройств
flutter devices

# Запуск на конкретном устройстве
flutter run -d <device-id>

# Запуск в Chrome (для веб-версии)
flutter run -d chrome
```

## Troubleshooting

Если приложение не запускается:

1. Проверьте версию Flutter:
```bash
flutter --version
```
Должна быть >= 3.0.0

2. Проверьте зависимости:
```bash
flutter doctor
```

3. Очистите кеш:
```bash
flutter clean
flutter pub get
```
