# Подключение к Yandex Cloud VM

## Информация о сервере:

- **IP:** 94.131.80.213
- **Логин:** kyte-777
- **SSH ключ:** Заказчик предоставил ключ `ssh-key-1765123214125`

## Вариант 1: Использование предоставленного SSH ключа

### Если заказчик дал файл ключа:

1. **Сохраните ключ** в файл, например: `C:\Users\1\.ssh\yandex_key`

2. **Подключитесь:**
   ```powershell
   ssh -i C:\Users\1\.ssh\yandex_key kyte-777@94.131.80.213
   ```

### Если заказчик дал публичный ключ для добавления:

Если заказчик дал вам публичный ключ для добавления в Yandex Cloud:
1. В консоли Yandex Cloud добавьте этот ключ в VM
2. Используйте ваш собственный приватный ключ для подключения

---

## Вариант 2: Создание нового SSH ключа

Если нужно создать новый ключ:

```powershell
# Создайте новый ключ
ssh-keygen -t rsa -b 4096 -f C:\Users\1\.ssh\yandex_cloud_key

# Покажите публичный ключ
cat C:\Users\1\.ssh\yandex_cloud_key.pub
```

**Попросите заказчика добавить этот публичный ключ в VM через консоль Yandex Cloud.**

---

## Вариант 3: Подключение через Yandex Cloud CLI

Если есть доступ к консоли Yandex Cloud:

1. Установите Yandex Cloud CLI
2. Подключитесь через CLI

---

## После успешного подключения:

Следуйте инструкциям в файле `DEPLOY_NOW.md`

# Если ключ в другом месте
ssh -i путь\к\ключу kyte-777@94.131.80.213

# Если нужен другой порт

---

## Если не получается подключиться:

1. **Проверьте что заказчик добавил ваш публичный ключ в VM**
2. **Попросите заказчика проверить группу безопасности** (должен быть открыт порт 22)
3. **Попросите заказчика дать вам приватный ключ** для подключения
